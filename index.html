🔄 MAC Changer

02 / NETWORK RECON ▸

📡 Network Scanner

ARP-based host discovery across subnets. Sends broadcast ARP requests and maps all live devices with their IP and MAC addresses in a clean table.

How I Built It
Built using Scapy — crafted ARP requests with pdst set to the subnet range, wrapped in an Ethernet frame with broadcast destination ff:ff:ff:ff:ff:ff. Used srp() to send and receive. Parsed the answered list to extract IP/MAC pairs and displayed them as a formatted table. Tested in a two-VM Kali/Windows 10 lab.

03 / MITM ATTACK ▸

⚡ ARP Spoofer + Detector

Poisons ARP caches of both target and gateway to intercept all traffic. Auto-restores ARP tables on exit. Companion detector script alerts on active spoofing.

How I Built It
Sent crafted ARP replies using scapy — telling the target I'm the gateway and vice versa. Enabled IP forwarding via /proc/sys/net/ipv4/ip_forward so traffic still flows. Used Python's signal module to catch Ctrl+C and restore the real ARP tables on exit. The detector sniffs ARP packets and compares sender MACs against known real addresses to flag anomalies in real time.

05 / MITM ATTACK ▸

🔀 DNS Spoofer

Intercepts DNS queries via Netfilterqueue and injects forged responses to redirect victims to an attacker-controlled IP. Integrates with iptables FORWARD rules.

How I Built It
Set an iptables FORWARD rule to redirect all DNS UDP packets to a Netfilter queue. Used netfilterqueue to intercept each packet, then scapy to check if it was a DNS query for the target domain. If yes, modified the DNS response to point to my IP, deleted the old checksum so Scapy recalculates it, then accepted the modified packet. The victim's browser then loaded my page instead of the real one.

MSc Dissertation

Outstanding-grade MSc research — built a complete PKI infrastructure from scratch and conducted a detailed comparative analysis of digital certificate tools.

★ Outstanding Grade
Evaluating and Comparing Digital Certificates by Implementing them in a Virtual Environment
University of Hertfordshire · MSc Cybersecurity · Module: 7COM1070

Digital certificates are the backbone of web security — authenticating servers, encrypting communications, and preventing identity fraud. This research evaluated two leading certificate management tools by building a real multi-layer PKI from scratch, deploying it on a live Apache web server, then comparing both tools across 12 technical dimensions.

Tools Compared
OpenSSL vs Java Keytool

Signing Algorithm

Deployment

Kali Linux VM
  • Created a Root Certificate Authority (CA) using OpenSSL with RSA 4096-bit keys — the trust anchor for the entire certificate chain.
  • Created an Intermediate CA signed by the Root — following real-world PKI best practice of never using the Root CA to sign end-entity certificates directly.
  • Generated server certificates, built a complete certificate chain, and deployed HTTPS on Apache. Imported the Root CA into browser trusted stores — verified in Firefox and Edge.
  • Repeated the full implementation with Java Keytool and compared both tools across algorithms, file formats (PEM vs PKCS12), SAN extension support, known vulnerabilities, and enterprise use cases.
// Conclusion

OpenSSL is superior for full cryptographic operations, scripting, and configuration flexibility — the right choice for enterprise PKI. Java Keytool excels at managing certificate stores within Java ecosystems. Both tools successfully secured HTTPS communications in the test environment, but OpenSSL offers far more control, automation potential, and library support.
